# 「ページ一覧」6枚 画像の趣旨メモ

対象: `stitch_yume_site1New/1._home/code.html` の「ページ一覧」セクション（6枚タイル）

## 目的（なぜ画像を置くか）
- 不動産サイトだと一番重要な「業種の即時理解」を、テキストだけでなく視覚でも担保する
- トップページ内のリズム（視線誘導・章の切り替え）を作り、単調さを減らす
- 画像は“主役”ではなく“空気感”として薄く敷き、読みやすさを最優先する（画像少なめ方針のまま）

## 6枚タイルで伝えたいこと（それぞれの絵の方向性）
1. Search（物件検索）
   - 街並み / 住宅外観 / 複数の選択肢が想像できる風景
2. Guide（案内の流れ）
   - 対面相談 / 内見の雰囲気 / 説明を受けているシーン（「同行・案内」の気配）
3. Strength（我々の強み）
   - “信頼感”のディテール（鍵・図面・書類など） or 落ち着いた内観（抽象でもOK）
4. Voice（お客様の声）
   - 暮らしの気配が出る内観（リビング/ダイニング等）＝「住んだ後」の想像ができる絵
5. FAQ（よくある質問）
   - 情報・サポートの連想（デスク/PC/書類/チャット的な絵）※写真は薄め表示が向く
6. Company（会社情報）
   - 店舗外観 / 受付 / オフィスっぽい内観＝「ここに相談できる」の安心感

## トーンの指針（TERASS寄せ）
- 明るい（ハイキー）・白い余白が多い・彩度低め（青み/グレー寄り）
- テキスト可読性が崩れる写真（暗い/コントラスト強すぎる/情報量が多すぎる）は避ける

## 実装の前提（デザインルール）
- 写真はタイルの背景に敷く（`background-size: cover` / `background-position: center`）
- 常時: 文字の下に薄いオーバーレイ（白寄り）を敷いて読みやすさを担保
- hover/focus: ほんのり暗くする＋微ズーム（派手にしない、0.2〜0.3秒程度）
